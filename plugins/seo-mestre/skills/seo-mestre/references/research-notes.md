# Notas de Referência — SEO MESTRE

Este pacote foi estruturado com base em princípios de fontes oficiais e boas práticas atuais.

## Bases conceituais

- SEO técnico continua sendo fundamento para experiências generativas.
- AEO/GEO/AIO devem ser tratados como extensões de SEO, não substitutos.
- Core Web Vitals medem experiência real de carregamento, interatividade e estabilidade.
- Lighthouse ajuda no diagnóstico de performance, SEO, acessibilidade e boas práticas.
- Dados estruturados devem seguir as diretrizes do Google e refletir conteúdo verdadeiro.
- `robots.txt` controla crawling, mas não deve ser usado como mecanismo para esconder páginas do índice.
- `llms.txt` pode ser útil como camada complementar para agentes, mas não é requisito oficial de Google Search.
- Crawlers de IA precisam ser configurados conscientemente conforme a estratégia do projeto.
